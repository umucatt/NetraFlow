# CHANGELOG

## 0.9.10

### 平台支持与下载

* 新增 macOS Apple Silicon 版本，使用 DMG 分发。
* 新增 Linux x64 版本，提供 AppImage 和 DEB。
* Windows 继续提供 x64 安装版和便携版，并统一三个平台的版本命名、图标和下载形式。
* 各平台采用符合系统习惯的窗口控制、启动、关闭和重新打开行为。

### 搜索与大数据体验

* 大幅提升大量账户和历史记录下的搜索打开、输入和查询速度。
* 优化中文输入法、连续输入和快速删除时的响应，减少结果区短暂空白、跳动或延迟更新。
* 搜索结果、预览、类别切换和数量显示更加稳定，键盘与鼠标选择保持一致。
* 优化大规模数据下的后台处理和内存占用，降低搜索对主界面流畅度的影响。

### 启动、窗口与锁定

* 优化启动阶段的主题、首屏和窗口响应，减少进入应用时的闪动和迟滞。
* Windows 和 Linux 可使用 `Ctrl+Shift+L` 锁定应用，macOS 可使用 `Command+Shift+L`。
* 未启用登录密码保护时，锁定操作会给出明确提示。
* 优化窗口激活、最小化、重新打开和重复启动时的行为，使不同平台下的使用体验更加一致。

### 数据与稳定性

* 完善 Windows、macOS 和 Linux 的本地数据与运行文件管理，继续将用户数据与缓存、日志等运行内容分开保存。
* 清除全部数据在三个平台使用一致流程，并只处理 NetraFlow 自身管理的本地数据。
* 优化启动、保存、关闭和异常状态下的数据衔接，提升长时间使用和频繁操作时的稳定性。
* 本版本延续现有正式数据格式，不需要用户手动迁移数据。

### 密码与交互

* 精简密码设置、修改和停用流程，使输入、强度提示和错误反馈更直接。
* 减少重复或无必要的提示，同一操作过程中的反馈更加一致。
* 优化锁定、解锁和密码相关对话框的布局与状态显示。

### 品牌与系统集成

* 统一应用内品牌图形以及各平台的应用图标。
* 改善任务栏、窗口、快捷方式、安装程序和锁定界面中的图标清晰度与一致性。
* 保留各平台原生窗口外观和系统交互习惯。


## 0.9.9

### 登录密码与本地数据

* 启用登录密码后，本地账本也会受到加密保护，不再只用于进入应用时的身份确认。
* 关闭登录密码保护后，本地账本会恢复为普通本地保存，已有账户、历史记录和设置不会受到影响。
* 加密快照不再使用单独设置的快照密码，新创建的加密快照统一使用创建时的登录密码。
* 修改或关闭登录密码不会自动改变以前创建的加密快照，恢复旧快照时仍需使用创建快照时的密码。

### 快照与安全设置

* 新增强制启用快照加密的设置，启用登录密码保护后，可避免误将快照以未加密方式导出。
* 调整登录密码与快照加密之间的联动，修改或关闭相关保护时会先进行必要确认。
* 优化密码设置、修改和关闭流程，密码强度提示、输入反馈和操作状态更加清晰。
* 移除原有的独立快照密码设置，减少重复的密码管理和容易混淆的安全选项。

### 数据保存与文件保护

* 加强本地账本的保存保护，发现账本文件在应用外被修改时，会停止直接覆盖并提示用户确认。
* 优化短时间内连续操作的保存方式，减少重复写入，同时确保最后一次修改能够正确保存。
* 优化保存失败后的处理，未成功写入的数据会继续保留在当前应用中，不会被错误标记为已经保存。
* 调整本地数据的保护范围，普通设置、运行状态和安全设置不会被混入核心账本。

### 启动与关闭

* 优化应用关闭前的保存处理，存在未保存修改或账本文件异常时，会先提供明确的处理选择。
* 选择继续保存时，会使用当前最新数据完成保存后再退出；选择不保存时，退出过程中不会再次自动写入。
* 安装版、便携版和开发版本同时启动时，只保留一个正在运行的 NetraFlow，并唤醒已有窗口。
* 优化核心账本无法读取时的启动提示，避免在数据状态不明确时继续进入应用。

### 界面与交互

* 调整本地账本与快照文件的异常提示，使问题对象、风险说明和可选操作更容易区分。
* 登录锁屏和安全设置中的密码错误改为在当前输入区域直接显示，减少额外弹窗。
* 优化安全确认弹窗的取消和返回行为，同时保留核心数据异常提示的强制确认规则。

## 0.9.8

### 数据与保存

* 将核心账本、普通设置、运行状态和安全设置分开保存，减少不同类型数据之间的相互影响。
* 修改主题、图表、备份和其他普通设置时，不再反复重写整个核心账本。
* 启动时发现核心账本损坏或无法识别时，会停止继续读取，避免异常数据被当作空白账本覆盖。
* 调整异常文件的处理方式，不再自动使用残留的临时文件或旧副本替换当前账本。

### 示例模式

* 示例模式改为使用完全独立的临时环境，示例账户、历史记录、设置和安全操作不再影响真实数据。
* 退出示例模式或下次启动应用时，会自动清理示例环境并恢复真实数据。
* 示例模式中不再执行自动快照，也不能导入或导出真实账本及用户配置。
* 优化示例模式与测试数据之间的隔离，避免临时内容进入真实账本目录。

### 快照与配置

* 核心快照只包含账户、分组和历史记录等账本内容，不再混入普通设置、运行状态和安全信息。
* 用户配置与核心账本采用独立的导入导出范围，恢复时更容易判断会影响哪些内容。
* 调整快照导入后的保存处理，避免导入过程中较新的数据被旧状态覆盖。
* 本地启动密码与快照密码在此版本中仍保持独立，本地启动密码不会加密核心账本。

### 修复与稳定性

* 修复闪记、汇总输入、余额修改、账户删除、归档、恢复和重命名等连续操作中，旧界面状态可能覆盖较新数据的问题。
* 修复部分延迟保存操作可能出现的数据顺序不一致问题。
* 优化开发运行与正式应用的数据目录隔离，避免读取或写入错误位置。

### 兼容性与说明

* 此版本调整了本地数据保存结构，不自动迁移此前开发版本遗留的旧数据文件。
* 当前 Windows 构建仍未启用代码签名和自动更新。
* 卸载时默认勾选删除本地用户数据，取消勾选后会按安装器现有规则保留数据。

## 0.9.7

### 数据保存

* 重做本地数据的整体保存流程，降低程序异常中断时产生不完整数据文件的风险。
* 分组、账户和历史记录等相关修改会作为一次完整操作保存，避免只写入其中一部分。
* 新数据只有在确认可以正常读取后才会替换原有文件，保存失败时会优先保留原数据。
* 修复数据整理或历史记录处理失败后，仍可能留下部分更新内容的问题。

### 异常保护与恢复

* 启动时发现本地数据损坏时，不再将其误认为首次启动的空白数据。
* 当前数据无法正常读取时，会尝试使用最近一次通过检查的保存内容恢复。
* 遇到来自更高版本或当前版本无法识别的数据时，会停止读取和保存，避免旧版本覆盖新数据。
* 保存完成后的最终检查失败时，不会继续使用异常文件。

### Windows 版本

* 完善安装版和便携版的构建与发布检查，降低发布产物缺失或内容异常的概率。
* 优化发布前的版本与文件检查，提高 Windows 发布包的一致性。
* 改进测试失败时的阻断处理，避免在检查未通过时继续生成发布版本。

### 兼容性与说明

* 现有 0.9.6 数据可以继续读取，并会在第一次实际保存时更新为当前版本使用的格式。
* 来自更高版本的数据会被安全拒绝，不会被当前版本修改。
* 当前 Windows 构建未启用代码签名和自动更新。
