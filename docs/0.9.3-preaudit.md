# NetraFlow 0.9.3 预审大纲

0.9.3 仍然是小版本，定位为优化、修正、瘦身和稳定性整理，不作为新增功能版本。

本文件只记录已确认的产品边界和后续审查方向，不要求立即实现所有条目。

## 1. 0.9.3 定位

0.9.3 关注：

- 优化
- 修正
- 瘦身
- 稳定性整理
- 清理旧结构
- 明确后续封装约束

0.9.3 不做大功能宣传，不把小修写成新增功能版本。

## 2. 卸载与用户数据处理原则

安装版、便携版以及未来可能出现的其他封装形式，均属于后续封装阶段议题。

当前阶段只讨论原则，后续封装阶段再实施。

卸载器后续只提供一个用户数据相关选项：

> 删除本地用户数据

该选项不附加额外备份提示、弹窗、挽留或二次确认。NetraFlow 在卸载阶段应保持克制，尊重用户卸载意图，不做越界干预。

程序残留清理仍然是目标。用户数据删除与程序残留清理应区分，但不通过复杂提示打扰用户。

## 3. 封装形态说明

安装版、便携版以及未来可能出现的其他封装形式，均属于后续封装阶段议题。

0.9.3 当前阶段不立即处理便携版卸载问题。便携版没有安装器意义上的卸载流程，不应套用安装版卸载逻辑。

## 4. 用户数据目录原则

无论后续采用安装版、便携版或其他封装形式，用户数据目录只允许保存用户主观形成的内容。

允许保存：

- 用户创建的数据
- 用户历史记录
- 用户修改过的配置
- 用户设置
- 用户密码保护配置
- 用户主动导入、创建、保存的数据

不允许保存：

- 默认配置
- 示例数据
- 临时数据
- 缓存
- 日志
- 可重建运行时文件
- 其他非用户主观形成的杂项

后续需要审查当前项目中是否存在默认配置、示例数据、缓存、日志或临时文件混入用户数据目录的情况。

## 5. 闪记旧结构清理

0.9.3 可继续审查并清理旧闪记结构：

- 不可达旧 stage
- 旧键盘分支
- 无实际流程的 stash / context / snapshot / touched 状态
- 旧修正页、暂存区、右键菜单等残留

但必须保留四种选区工具：

- 单选
- 交集
- 合集
- 删除

四种工具属于设计保留项，不应被当作旧复杂功能删除。

## 6. 四种选区工具文案

主文案统一为：

- 单选
- 交集
- 合集
- 删除

解释性描述可以放在 title 或 aria-label 中，但主 UI 不应混杂“拖选 / 合并 / 相减”等多套表述。

## 7. 汇总导入重复识别

汇总导入重复识别只用于提示风险，不阻断导入。

汇总导入是录入来源，快照是完整数据备份/恢复。两者性质不同，不应为了重复风险提示而把 importedHashes 强行纳入快照逻辑。

0.9.3 只确认现状，不扩展快照结构，不做批次撤销。

## 8. 登录密码

登录密码文案保持现状。0.9.3 不新增说明文案，不做整体数据加密。

整体加密属于后续独立议题，需要综合实现难度、迁移成本和用户体验后再讨论。

## 9. App.tsx 瘦身

0.9.3 可以继续审查 `App.tsx`，但不一次性大拆。

推荐顺序：

1. 死代码清理
2. 纯工具 / 纯 service 抽离
3. 小 controller 抽离
4. 最后再考虑复杂状态拆分

暂不优先拆全局弹窗状态、右侧面板状态等交织较深的部分。

## 10. 本轮不要做

- 不处理便携版卸载
- 不把 importedHashes 纳入快照
- 不添加登录密码说明文案
- 不做批次撤销
- 不做整体数据加密
- 不重构搜索架构
- 不大拆 App.tsx
- 不删除闪记四种选区工具
- 不把 0.9.3 写成新增功能版本
